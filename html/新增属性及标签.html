# HTML5元素新增

# 语义元素

header 为文档或节规定页眉，应该被用作介绍性内容的容器。

nav 定义导航链接集合。

section 定义文档中的节。

aside 页面主内容之外的某些内容。

article 规定独立的自包含内容。应用场景：论坛、博客、新闻。

main 规定文档的主内容。

footer 为文档或节规定页脚。通常包含文档的作者、版权信息、使用条款链接、联系信息等。

# 图片元素

# picture

用于创建图片响应式，元素显示方式inline。会根据屏幕匹配的不同尺寸时显示不同图片。通常会内嵌零或多个<source>元素和一个<img>元素。每个<source>元素都会匹配不同的设备并引用不同的图像源，如果没有匹配的，就默认使用<img> 。

<!-- 当设备最小尺寸大于465px 显示url2图片；最小尺寸大于650px显示url1图片，否则显示url3图片 -->
<picture>
  <source type="image/svg+xml" media="(min-width: 650px)" srcset="url1">
  <source media="(min-width: 465px)" srcset="url2">
  <img src="url3">
</picture>

TIP

source 元素通常有两个属性。media 属性用于当前源文件生效的媒体查询条件，查询结果为false时跳过当前source元素；type属性用于设置元素的srcset属性指向的资源的MIME类型。如果用户代理不支持指定的类型，那么这个 <source> 元素会被跳过。

除此之外，要想在不同分辨率的设备显示相同尺寸的图片可以这样(也就是我们常说的点语法)：

<img srcset="1倍图.jpg,
             2倍图.jpg 2x,
             2倍图.jpg 3x"
     src="2倍图.jpg" alt="图片信息提示">

# figure

常用于图片的容器，其表现类似于 div。

  • figcaption 该标签常与figure标签共同出现，用于定义图片的标题。除了让页面结构更有意义，没有特别的样式（文本没有任何样式上的变化）如下实例：
<figure>
  <img src="img_pulpit.jpg" alt="The Pulpit Rock">
  <figcaption>这是上面图片的标题</figcaption>
</figure>

# 表单元素

<datalist> 为 <input> 元素规定预定义选项列表。默认子项不会显示处理，会在输入数据时看到预定义选项的下拉列表（下拉列表子项由 <option> 设置）。<input> 元素的 list 属性必须引用 <datalist> 元素的 id 属性。

 <input list="fruits" />
    <datalist id="fruits">
      <option checked value="香蕉">
      <option value="苹果">
      <option value="橘子">
    </datalist>

# video 视频

autoplay 如果出现该属性，则视频在就绪后马上播放。属性值为 Boolean 类型。

controls 如果出现该属性，则向用户显示控件，比如播放按钮。属性值为 Boolean 类型。

loop 如果出现该属性，则当媒介文件完成播放后再次开始播放。属性值为 Boolean 类型。

muted 如果出现该属性，视频的音频输出为静音。属性值为 Boolean 类型。

poster 属性用于指定视频下载时显示的图像，或者在用户点击播放按钮前显示的图像。属性值为 url 。

src 属性用于指定视频播放的地址。如果想在所有浏览器播放视频文件 ，可在 <video> 元素使用 <source> 元素。例如

<video width="320" height="240" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.ogg" type="video/ogg">
    您的浏览器不支持 video 标签。
</video>

# 表单输入类型

type

color 主要用于选取颜色，设置 type 属性值为 color 后，会弹出颜色选择器供用户选取。可设置value属性设置颜色选择默认值，但其value值必须时十六进制颜色值（如 #ff000）。

date 允许你从一个日期选择器选择一个日期 格式为 X年/X月/X日。可设置value属性设置日期选择器默认值，但其value值格式必须 yyyy-mm-dd。

email 用于应该包含电子邮件地址的输入字段,在提交表单时，会自动验证用户输入是否合法有效。

month 允许你选择一个月份。

number 用于应该包含数值的输入域。常与 >>max、min（规定允许的最大、小值）、step属性配合使用（规定输入字段的合法数字间隔）。

range 用于应该包含一定范围内数字值的输入域。显示为滑动条。可设属性与number类似。

search 用于搜索字段。

tel 用于应该包含电话号码的输入字段。可设置minlength或者maxlength限制输入电话号码的长度。

time 允许你选择一个时间。

url 用于应该包含 URL 地址的输入域。在提交表单时会自动验证是否合法。

week 允许你选择周和年。浏览器会自动计算该天属于这一年的第几周。

# 新表单属性

<form> 新属性

autocomplete 规定表单或输入字段是否应该自动完成。当开启时，浏览器会基于用户之前的输入值自动填写值。属性值为 on/off 。适用于一下表单输入类型：text, search, url, telephone, email, password, datepickers, range 以及 color。

<input> 新属性

autofocus 如果设置，则规定当页面加载时 <input>元素应该自动获得焦点。属性值为Boolean类型。

formaction 规定向何处提交表单数据，该属性会覆盖<form> 元素中的action属性。

formenctype 属性描述了表单提交到服务器的数据编码 (只对form表单中 method="post" 表单)，该属性覆盖 form 元素的 enctype 属性。该属性与 type="submit" 和 type="image" 配合使用。

formmethod 属性定义了表单提交的方式。该属性会覆盖 <form> 元素的 method 属性。该属性可以与 type="submit" 和 type="image" 配合使用。

formtarget 属性指定一个名称或一个关键字来指明表单提交数据接收后的展示。该属性会也会覆盖 <form>元素的target属性。该属性与type="submit" 和 type="image"配合使用。

multiple 属性是一个 boolean 值。该属性规定<input> 元素中可选择多个值。属性适用于以下类型的 <input> 标签：email 和 file。

pattern 属性描述了一个正则表达式用于验证 <input> 元素的值。注意:pattern属性适用于以下类型的 <input> 标签: text, search, url, tel, email, 和 password。

placeholder 属性提供一种提示，描述输入域所期待的值。简短的提示在用户输入值前会显示在输入域上。注意: placeholder 属性适用于以下类型的 <input> 标签：text, search, url, telephone, email 以及 password。

required 属性是一个 boolean值。规定必须在提交之前填写输入域（不能为空）。注意:required 属性适用于以下类型的 <input> 标签：text, search, url, telephone, email, password, date pickers, number, checkbox, radio 以及 file。
