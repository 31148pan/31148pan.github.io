# HTML标签分类

html 按表现形式可分为三大类。

# 块级元素

# 块级元素的特点

1、每个块级元素会独占一行，从上到下排布。
2、可以直接控制盒子模型的相关css属性。

TIP

默认情况下，块级元素的宽度是它父级元素的宽度/高度是它本身内容的高度。

# 常用的块元素

  • <div> 该标签常用做块级布局容器。

  • <p> 段落标签，默认情况下，会在元素前后添加一个额外的空行（默认为浏览器字体大小16px），标题元素h标签也是如此。

  • <h1> ~ <h6> 标题标签，一个页面只能有一个 h1 标签，标题级别一般使用到三、四级。

  • <table> 表格。

  • <hr> 水平线。

  • <ul>、<ol>、<dl> 无序、有序、自定义列表 。

ul与ol,都有相同的列表子元素和list-style（list-style-type、list-style-image及list-style-position的缩写）属性。
ol 标签特有属性：reversed（规定列表顺序为降序。9,8,7...）、start（number 类型，规定有序列表的起始值）及type（五种可能值：1、A、a、I、i，用于在列表中使用的标记类型）。开发中应该尽可能使用标签代替。

  • <form> 表单。

1、action 属性规定当提交表单时，向何处发送表单数据。

2、method 属性规定如何发送表单数据，即采用什么方式提交表单数据.

3、name 属性规定表单的名称。对当前表单或者表单控件进行标识。

# 行内元素

# 行内元素的特点

1、多个内联元素会从左到右在一行显示，排不了会换行。

2、不能直接控制宽度、高度以及行高属性，但是可以直接设置内外边距的左右值,设置上下内外边距值时浏览器可能会忽略错误解析。

TIP

内联元素的宽是由文本内容的宽度决定，高度一般是由文本字体大小决定。

# 常用的行内元素

  • <a> 锚点，为文本或者图像定义超链接。

1、不跳转（跳转到当前页面）设置属性 href="#"或 href="javascript:"。
2、跳转到另外的页面，直接填写另一个页面的 url 即可。
3、在当前页面进行定位（先设置 a 标签的 href 属性为"#id名"，再在页面的目标位置加入一个目标标签（可以是任意标签），并给这个标签设置 id 名。
4、进行下载：<a href="目标文件的路径">下载</a>（不推荐使用）。
5、标签的 target 属性，规定在何处打开链接文档。例如 target="_blank "表示在新窗口打开链接，默认_self。其他属性值有 _blank 、_top 、 _parent。

  • <span> 常用 文本 或 图标 容器。
  • <b>：加粗 <u>:下划线 <i>: 倾斜 <sub>: 删除线（建议使用对应的语义标签<strong>、<ins>、<em>、<del>）。这些文本格式化标签多用于创建小图标的容器）。
  • <br> 定义换行。
  • <sup>、<sub> 定义上下标。如 x2、y2。
  • <label> 标识作用，多用在用户界面中的项目的标题(可以设置盒模型相关属性)，与 <input> 一起使用。

将元素 <label> 与 <input> 元素相关联(可以使辅助技术用户更容易理解应输入哪些数据，单击关联的标签时可以自动获取焦点或者激活输入)，只需要提供 <input> 一个 id 属性。再设置 <label> 的 for 属性值是相同的输入的属性 id 。或者，可以将 <input> 元素直接嵌套在内部 <label> ，则可以不需要设置 for 和 id 属性，形成隐式关联。


    <label for="searchText">
      请输入：
      <input type="text" id=searchText>
    </label>
    <!-- 或者 -->
    <label for="searchText1">请输入：</label>
    <input type="text" id="searchText1">

  • <select> 创建单选或多选菜单（可以设置盒模型相关属性）。

value 用于设置提交表单时传入服务器时所选中的值。
name 属性，规定下拉列表的名称。用于对提交到服务器后的表单数据进行标识。以下为例提交表单时会将 fruits=apple 提交到服务器。
selected 属性，用于设置下拉列表 option 的默认选项，例如：

      <option value="banana">香蕉</option>

    </select>

  • <video> 、<audio> 视频、音频。虽然都是行内元素，但 video 可以控制宽高等属性、audio 不能控制宽高，却能够独占一行。
  • <img> 向网页中嵌入一幅图像。

alt 必须属性，规定图像的替代文本；src 属性，规定显示图像的 URL。URL地址可以是相对路径（相对于当前项目的路径）或者绝对路径（带盘符的完整路径）。
width、height，可选属性，用于设置图像的宽度和高度。不设置宽高时图片会以默认大小展示。

  • iframe 用于设置网页的内联框架。

src 属性用于设置框架的链接地址，可为绝对路径或者相对路径。设置此属性后会将该链接的网页内嵌到当前标签处。
name 用于设置当前框架的名称。
width、height 用于设置框架的宽高。

<iframe src="url">

# 行内块元素

# 行内块元素特点

1、可以与其他行内元素或者行内块元素共处一行。
2、可以设置 css 盒模型相关属性。

# 常用的行内块元素

  • <button> 创建一个可点击的按钮，需正确闭合标签（可以设置盒模型相关属性）。

  • <input> 用于搜集用户信息

type 属性定义单行的输入字段，用户可在其中输入文本。默认宽度为 20 个字符。

1、password 属性用于定义密码字段。该字段中的字符被掩码。
2、radio 定义单选按钮。
3、checkbox 定义复选按框。
4、button 定义可点击按钮（多用于通过 JavaScript 启动脚本）。
5、reset 定义重置按钮。重置按钮会清除表单中的所有数据。
6、submit 定义提交按钮。提交按钮会把用户在表单中的数据发送到服务器。
7、image 定义图像形式的提交按钮，必须与 src 属性 和 alt 属性结合使用。
8、file 文件上传。默认只会上传一个文件可设置multiple属性，即可多文件上传。同时可设置accept属性限制传入的数据类型。例如：

accept="image/png" 或 accept=".png" — 只接受 png 图片。以逗号分开即可设置多种类型文件。如 accept="image/png, image/jpeg" 或 accept=".png, .jpg, .jpeg"。设置image/*，即为任意图片类型。
accept=".doc,.docx,.xml,application/msword,application/vnd.openxmlformats-officedocument.wordprocessingml.document" — 接受任何 MS Doc 文件类型.
如果想限制某一类型的文件上传可以这样 accept=".后缀名"。

TIP

如果想要获取所选中的文件信息，可通过HTMLInputElement.files属性获得。该属性返回一个FileList对象。 之后，可通过 FileReader 对象读取所选择的文件信息。

checked 属性，用于是否在页面加载时应该被选定复选框或单选按钮。用法：在需要设置的标签上设置 checked="checked" 即可。

disabled 属性，是否禁用 input 元素。用法：在需要设置的标签上设置disabled="disabled"即可。

value 属性,在不同的 input 表单输入类型上有不同的表现。

1、type="button", "reset", "submit" 定义按钮上的显示的文本
2、type="text", "password", "hidden","" 定义输入字段的初始值。
3、type="checkbox", "radio", "image" 定义与输入相关联的值，checkbox 与 radio input 类型必须设置 value 属性。 ...

name 属性，规定 input 元素的名称。用于对提交到服务器后的表单数据进行标识，只有设置了 name 属性的表单元素才能在提交表单时传递它们的值。

alt 属性，为用户由于某些原因无法查看图像时提供了备选的信息。只能与 <input type="image"> 配合使用。

# HTML全局属性

所谓全局属性就是所有html标签都可以设置。常见的有：
1、class（用于设置元素类名，允许设置多个类名）。
2、id（用于设置元素唯一id标识，只能设置一个）。
3、style（用于设置元素行内样式，规则是 key:value,多个属性以;分开）。
4、title（用于设置鼠标hover时元素的提示信息）。
5、lang（用于设置元素的语言类型，由于通常只会设置在html标签上）。
6、draggable（用于设置元素是否可以拖动）。
7、data-*（用于存储页面的私有数据）.
8、hidden （用于设置元素是否显示，值为Boolean类型）。
