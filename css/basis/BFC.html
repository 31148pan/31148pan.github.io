# BFC

# 什么是BFC?

BFC(Block formatting context)，块格式化上下文是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。它是一个独立的渲染区域。形成 BFC 的盒子其内部与外部元素之间不会相互影响。而且一个元素不能同时存在两个 BFC 中。

# 如何形成BFC？

  • 浮动元素（元素的 float 不是 none）。

  • 行内块元素（元素的 display 为 inline-block）。

  • 绝对定位元素（元素的 position 为absolute 或 fixed）。

  • 表格单元格（元素的 display为 table-cell，HTML表格单元格默认为该值）、表格标题（元素的 display 为 table-caption，HTML表格标题默认为该值）

  • overflow 值不为 visible 的块元素

  • 弹性元素（display为 flex 或 inline-flex 的子项目）。

  • 网格元素（display为 grid 或 inline-grid 元素的直接子元素）。

# BFC的应用

  • 解决margin合并现象。当相邻的子元素都设置水平或者垂直方向的 margin 后会发生重叠，取值为两者之间最大的值。这是我们可以令其父元素或者本身形成 BFC 解决。

  • 解决margin塌陷现象。当父元素内包含子元素，给子元素设置 margin-top或者margin-bottom 父元素会跟着一起掉下去。这里除了让其父元素形成 BFC 之外还可以为其父元素添加height、border、padding、文字阻隔margin，就不会发生塌陷。

  • 清除浮动的影响。当我们设置元素浮动时，由于设置了浮动，所以该元素就形成了 BFC。由于 BFC 的隔离作用，所以父元素可能会（没设置宽高）造成塌陷，这时我们会让其父元素形成 BFC ，而重新撑开父盒子（BFC计算高度时，会考虑 BFC 中所有的元素，包括 其它的 BFC）。对于后面的元素，考虑到会被浮动元素盖住。所以，我们会在浮动后面的元素上设置 clear 属性（注意不要在浮动元素上清除浮动，因为浮动元素脱离标准流，清除浮动没意义） 或者创建 BFC 清除浮动。
