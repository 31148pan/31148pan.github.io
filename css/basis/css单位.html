# css单位

CSS中通常会有以下几种单位：

  • px css像素单位，是一种绝对单位。相对于设备屏幕的分辨率。屏幕分辨率越高，渲染同一面积的像素点越多也就越清晰。

  • em 相对单位，通常可以理解为相对于当前元素的父元素的font-size，由于继承性，1em 会一层一层往上找，而浏览器默认字体大小为16px,所以默认1em=16px。

  • % 相对单位，根据属性不同参照物就不同。

1、在position 定位中，其值参照它的包含块（即可能不是父元素）,因为定位不同属性（relative、absolute等）的参照物也不同。

2、在 盒模型 中，width、min/max-width属性的值为百分比时，其相对于包含块的width进行计算;height、min/max-height属性的值为百分比时，其相对于包含块的height进行计算（这里有个特殊，如果包含块是根据内容自适应的，则设置此值无效，比如块级元素，默认情况下高度有内容决定，宽度为包含块决定此时设置百分比高度则无效）；padding和margin相对来说更为复杂一些，如果书写模式（writing-mode）是水平的，则相对于包含块的width进行计算；如果书写模式是垂直的，则相对于包含块的height进行计算。

3、font-size 使用百分比是基于父元素中font-size进行计算。

4、text-align和padding有点类似，和书写模式有一定的关系。如果书写模式是水平的，则相对于width进行计算，如果是垂直的，则相对于height进行计算。

5、line-height 基于自身font-size进行计算。

6、vertical-algin则基于自身line-height计算。

7、border-width边框属性是不支持%单位的，但在border-radius边框圆角和border-image-width边框图片两个属性上是可以使用百分比为单位的。如果在border-radius中使用百分比单位，也就是说圆角的半径是通过百分比来进行计算的，即：水平方向的半径是相对于元素width计算，垂直方向的半径是相对于元素高度进行计算。

8、translateX()的百分比相对于元素自身的width计算；translateY()的百分比相对于元素自身的height计算。

9、background-size 也是可以使用百分比，相对于包含块。

  • rem 相对单位，始终相对于根元素html/body的font-size。默认情况下1rem = 16px。所有只要设置body/html的fontsize=62.5%，即可当前元素1rem = 10px.

  • vw, vh: 视窗相对单位，分别是视口宽度的1/100和视口高度的1/100。对应的还有vmax和vmin属性。

  • deg 角度单位。通常用于CS3 transfrom属性中。

  • s、ms 时间单位。通常见于CS3transition、animation及JS方法setTimeout及setInterval的第二个属性中。

注意：单位与长度之间不能有空格，像 font-size:16 px 这样浏览器是无法正常解析的。
